Built for operations teams

Everything on one dashboard: live probes, credential handling, and launch actions tuned for Windows-heavy environments.

Fleet intelligence

  • Live WinRM/PowerShell probes for CPU, RAM, disks, services, firewall profiles, adapters, and uptime.
  • Inline search with highlighting across names, notes, services, OS type, and recent errors.
  • Reorderable cards with saved layout, plus quick filters for high CPU, RAM, disk, reboot-needed, or service issues.

Credential-aware launch

  • RDP on double-click with stored credentials; reuses existing TERMSRV profiles and per-host overrides.
  • Global credential is shared across probes and RDP; host-specific credentials override when present.
  • Linux hosts (set OsType=Linux) use SSH probes and skip RDP launch automatically.

Operator-focused UX

  • System tray access with autostart toggle, theme selection, and quick refresh controls.
  • Bounded-concurrency refresh to handle large lists without locking the OS; throttled quick probes when you interact.
  • Dedicated error view, debug logging switch, and light/dark modes that respect system defaults.

What QuickProbe inspects

Consistent dashboards for mixed fleets. Windows uses WinRM and PowerShell remoting; Linux uses SSH with the same visual patterns.

Windows hosts

  • CPU, memory, process counts, uptime, and reboot-needed signals.
  • Disk inventory with free space alerts and service state tracking with templated service lists.
  • WinRM listeners, firewall profiles, and active network adapters.
  • Per-host notes and groups that are searchable and visible on the cards.

Linux hosts (SSH)

  • CPU and memory load plus process count visibility.
  • Disk usage and service state checks aligned to the same templates.
  • Uptime, distro info, and quick identification in the dashboard.
  • Tagged via OsType=Linux in hosts.csv to skip RDP actions.

Workflow in four steps

Built for daily use by Windows administrators who need fast visibility and instant launch actions.

1. Authenticate once
Store a global credential in Windows Credential Manager (QuickProbe:Default).
2. Load your hosts
Manage %APPDATA%\QuickProbe\hosts.csv or edit via the host manager (groups, notes, service templates).
3. Probe and filter
Run full refresh or per-card quick probes; search highlights matches across all card content.
4. Launch with context
Double-click to RDP with credential reuse; SSH probes for Linux hosts preserve the same dashboard feel.

Data stays on your machine

QuickProbe keeps everything local using Windows-native storage. No cloud dependencies.

Data type Location
Global credential Windows Credential Manager (QuickProbe:Default)
Per-host credentials Windows Credential Manager (QuickProbe:HOST/<SERVER>)
RDP profiles Windows Credential Manager (TERMSRV/...) reused for launches
Hosts list %APPDATA%\QuickProbe\hosts.csv
RDP connection files %APPDATA%\QuickProbe\Connections\
Settings and state %APPDATA%\QuickProbe\settings.json and localStorage (qp_settings)
Debug logs %APPDATA%\QuickProbe\QuickProbe_Debug.log
