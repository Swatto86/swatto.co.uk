QuickProbe

Windows Server health probing and RDP launchpad

QuickProbe is built for Windows System Administrators who need quick, repeatable answers: is the host reachable, are the right services up, and is it ready for an RDP session. Linux hosts can be included for limited probe-only visibility, without Windows-specific launch actions.

Designed for Windows System Administrators

One screen for health signals and launch actions, optimised for Windows Server estates.

Fast fleet visibility

  • At-a-glance status across your host list so you can spot issues quickly.
  • Search and filters to find the right server fast, with layout preferences retained locally.
  • Per-host refresh to spot-check a system without interrupting the rest of the dashboard.

Health signals that matter day-to-day

  • Reachability and responsiveness checks designed for practical triage.
  • Key service visibility to confirm expected workloads are running.
  • Notes and grouping so operational context sits next to the host.

RDP launch with fewer prompts

  • Launch RDP quickly using stored credentials where appropriate.
  • Optional per-host overrides remain isolated from your global default.
  • Linux-tagged hosts remain probe-only and do not offer Windows launch actions.

What QuickProbe inspects

Windows-first signals with consistent layout across mixed fleets.

Windows hosts (primary)

  • Core health signals across CPU, memory, disks, and expected services.
  • Connectivity and “ready to manage” posture to support rapid triage.
  • Per-host notes and grouping to keep ownership and context clear.

Linux hosts (limited probe support)

  • Probe-only visibility into availability and selected system signals.
  • Consistent card layout so Linux hosts fit naturally into the same dashboard.
  • No RDP actions exposed; Linux entries are intentionally “view and verify”.

Workflow in four steps

A simple loop: set context, probe, filter, launch.

1. Set credentials
Choose a default credential for probes and RDP launches, with optional per-host overrides.
2. Load hosts
Add servers and notes so the dashboard reflects the estate you actually manage.
3. Probe and filter
Refresh as needed; search and filters help you isolate the server you care about.
4. Launch
Start an RDP session directly from the host card when the server is in a good state.

Local-first data handling

Designed to operate without external service dependencies.

  • Credentials are stored using Windows Credential Manager (no cloud storage).
  • Host lists, settings, and connection artifacts remain on the device.
  • No telemetry is sent to third-party services by default.
